## 🎯 一句话讲清楚

**AI Drama Production** 是一个 [Claude Code](https://docs.anthropic.com/en/docs/claude-code/skills) 技能（skill），它把你的粗糙创意变成一份完整的 AI 短剧、AI 短片、预告片、动漫图轻动作视频制作方案。

它不生成视频，而是生成**可生产蓝图**：剧本、分镜、参考图 prompt、单片段视频 prompt、桥接镜头、保险插入镜头、修复 prompt、声音设计、连续性管理和剪辑装配指南。你的工作重点可以放在前期想法和审核，提示词、设定、桥接、打磨交给 Skill。

## 🎬 一个入口：电影生产线

| 能力 | 适合什么 | 产出什么 |
|---|---|---|
| **电影模式** | 真人感 AI 短片、电影感短剧、预告片、比赛作品、动漫图生视频、静态图轻动作 | 分镜表、Join Contract、单片段视频提示词、桥接镜头、保险素材、修复提示词 |
| **快速提示词包** | 只需要小段测试、角色/场景参考图、分镜 prompt 或短 AI 漫剧片段 | 精简输出：假设、生产锁、参考图、分镜、`VIDEO_MAIN`、高风险桥接和审查 |
| **完整制作圣经** | 需要完整单集、多集、商业交付或系统化制作文档 | 完整输出：剧本迭代、四基石、角色/场景/道具、风格、分镜、剪辑、声音、修复 |
| **复制粘贴提示词包** | 你只想负责创意和审核，不想自己补桥接和修复方案 | 默认每段只复制 `VIDEO_MAIN`；风险处才追加 `VIDEO_BRIDGE` / `VIDEO_INSERT` / `VIDEO_REPAIR` |
| **动漫图轻动作变体** | 已有动漫图片，只需要简单推拉摇移、表情/环境轻动、剪映关键帧 | 仍走电影模式，用更轻的运镜、更短动作 beat、更强参考图锁定 |

> 旁白驱动、动漫图轻动作、静态图动画，都按电影模式处理，只是降低运动复杂度。

## ✨ 为什么需要用这个技能？

大多数 AI 视频项目死在**生成之前**：不是工具不行，而是镜头设计、连续性、提示词结构和后期补救方案不完整。

这个技能重点解决四个真实痛点：

1. **有想法但不知从哪下手** — 接入、四基石、剧本讨论、导演讲戏、分镜、提示词包，一路生成到可执行文档。
2. **角色每镜长得不一样** — 先用干净三视图锁定脸、身形、服装和道具，再用完整角色设定板补表情、细节、日常/工作/剧情状态；每个视觉名词都必须能追溯到 `@图片` 或 `@视频`。
3. **镜头接不上** — `Join Contract` 逐个审查相邻镜头：前镜结束状态、后镜开始状态、状态差异、风险等级、是否允许硬切、桥接镜头、声音桥和失败备用剪法。
4. **AI 生成失败后不会修** — 每个高风险镜头预置桥接、插入、修复 prompt，让你可以直接复制粘贴重试，而不是临场想办法。

## 🚀 功能一览

| 功能 | 说明 |
|---|---|
| **低输入默认值** | 最多追问类型、主角关系、第一集钩子、结尾悬念、目标平台；其余由 Skill 补齐 |
| **分层剧本开发** | 快速提示词包用三步短审查；完整制作圣经才展开 3-5 轮多 Agent 讨论 |
| **导演讲戏** | 把抽象情绪转成具体动作、镜头、光线、声音和剪辑点 |
| **两阶段角色参考图** | 先生成白底三视图/六视图锁定角色，再生成完整角色设定板，包含肖像、三视图、表情、细节、穿搭、道具和剧照氛围 |
| **参考图优先设计** | 角色、场景、道具先出参考图 prompt，再写视频 prompt |
| **生产锁** | 先固定角色、场景、风格、镜头、连续性、声音和禁止漂移，所有视频提示词继承同一套约束 |
| **Shot State Contract** | 每条 `VIDEO_MAIN` 先锁首帧、屏幕方位、主体/道具状态、镜头覆盖模式、动作变化和终帧，减少初始状态错误 |
| **时间轴分镜提示词** | `VIDEO_MAIN` 按固定资产、风格质感、画面动作三层写，并用 `0-1.5s` 等时间段描述可见动作 |
| **Join Contract** | 每两个相邻镜头都有连接合同，禁止高风险硬切 |
| **桥接镜头自动生成** | 换地点、抱人/救人、姿态跳、双人互动、道具状态变化等自动补 `VIDEO_BRIDGE` 或 `VIDEO_INSERT` |
| **复制粘贴提示词包** | 默认每个正常片段输出一条可直接使用的 `VIDEO_MAIN`，风险处再补桥接、保险、修复 prompt |
| **内嵌声音** | 视频 prompt 内直接写 `[对白]`、`[音效]`、`[配乐]`，不创建 `@音频` 引用 |
| **轻量粗剪优先** | 用已有参考图、缩略图或占位图审查节奏、构图、地点和对白时长，默认不走额外关键帧流程 |
| **后期修复循环** | 用户提供生成片段后，继续审查断裂点并输出替换桥接/插入 prompt 和剪辑决策表 |

```text
💡 接入
  → 🏗️ 四基石（世界观 / 角色 / 故事 / 风格）
  → 📝 分层剧本开发（快速短审查 / 完整多 Agent）
  → 👤 角色三视图锁定 + 完整设定板
  → 🎬 导演讲戏
  → 📊 分镜 + Join Contract + 高风险桥接
  → 🖼️ 轻量粗剪检查
  → 📋 单片段单提示词包
  → 🎥 视频生成批次
  → 🧩 保险素材 / 修复提示词
  → 🔊 声音与剪辑
  → ✅ 审查与二次修复
```

## 🔧 工具链

| 工具 | 用途 |
|---|---|
| **Seedance / 即梦 / 其他图生视频工具** | 生成主镜头、桥接镜头、插入镜头 |

> 生成的 Markdown 可以脱离 Claude Code 独立使用。按文档里的提示词、装配指南和审查清单执行即可。

## 🧠 核心设计原则

| 原则 | 为什么重要 |
|---|---|
| **故事优先** | 前提、冲突和情感决定观众留存 |
| **先锁角色，再做氛围** | 主角先用干净三视图锁定身份，再用完整设定板扩展表情、服装变体、道具和情绪场景 |
| **先锁生产约束，再写镜头** | 生产锁让每条视频提示词继承同一套角色、场景、风格、镜头、连续性和声音规则 |
| **先锁首帧和终帧，再写动作** | Shot State Contract 让模型知道一开始画面是什么、动作改变什么、最后停在哪里 |
| **先拆时间，再写提示词** | 时间轴动作能约束模型只完成本镜头该做的事，减少动作过密和镜头乱跳 |
| **先锁参考，再写视频 prompt** | 角色、场景、道具一致性必须前置 |
| **先粗剪，再生成视频** | 静态图粗剪能低成本发现断裂 |
| **先审接头，再出片** | AI 视频没有记忆，相邻镜头必须单独设计 |
| **高风险不硬切** | 地点、姿态、道具、双人互动变化需要桥接或插入 |
| **声音嵌入 prompt** | 对白、音效、环境和音乐决定剪辑节奏 |
| **失败也要可修** | 每个高风险镜头都要有修复 prompt 和备用剪法 |
